# LSTM

由Hochreiter 和 Schmidhuber（1997）介绍引入。 不同于RNN单元中只是一个简单的tanh激活函数。 LSTM则通过门控的思想过滤信息, 比如σ∈[0,1]\sigma \in [0,1]σ∈[0,1]选择筛选信息量

  • 假设这个任务是一个基于上文预测最后一个词的语言模型
  • 上一个词的隐态和当前词, 确定要不要遗忘当前主题Ct−1C_{t-1}Ct−1​的信息

  • 第一个输入门iti_tit​决定哪些数据需要更新
  • 一个tanh⁡∈[−1,1]\tanh \in [-1,1]tanh∈[−1,1], 生成新的主题, 然后用σ\sigmaσ去控制要不要把新的语言主题信息加入到单元状态中, 以替代我们要遗忘的主题
  • 新的单元状态 = 遗忘旧的状态 + 新的及意义信息

    Ct=ftCt−1+itC~tC_t = f_tC_{t-1} + i_t\tilde C_t Ct​=ft​Ct−1​+it​C~t​

  • 最后把新的单元状态, 再通过一个tanh⁡\tanhtanh, 把输出限制在[−1,1][-1,1][−1,1], 然后和σ\sigmaσ的权重相乘, 得到下一个隐态

# LSTM如何防止梯度消失和梯度爆炸

由于导数的链式法则, 如果导数都是小数或者都大于111, 则会使得总体度消失或者爆炸, 如果只是梯度爆炸, 还可以通过gradient clipping限制范围的方法来解决,

但梯度消失则没办法, 而LSTM中的单元状态St=∑τ=1tΔSτS_t = \sum_{\tau = 1}^t \Delta S_{\tau}St​=∑τ=1t​ΔSτ​, 他的导数不再是乘积的形式, 就可以避免了

# GRU

循环门单元（Gated Recurrent Unit）由 Cho, et al. (2014)提出。 主要是简化LSTM单元的结构: 将遗忘和输入合并到一个门, 并且合并了单元状态和隐态

  • 这里的候选隐藏层(candidate hidden layer) h~t\tilde h_th~t​ 和 LSTM中的 C~t\tilde C_tC~t​ 类似, 可以看成当前时刻的新信息
  • GRU没有输出门(把CtC_tCt​单元状态整个一行给拿掉了), 通过候选和前一时刻的隐藏层得到这一时刻的隐藏层信息

    ht=(1−zt)ht−1+zth~th_t = (1-z_t)h_{t-1} + z_t\tilde h_t ht​=(1−zt​)ht−1​+zt​h~t​

  • 一般来说短距离相关的单元重置门活跃(rt=1,zt=0r_t=1, z_t=0rt​=1,zt​=0就相当于RNN了), 而长距离依赖的单元更新门活跃
